## 💡 Ключевые решения дизайна

### 1. Event-Based Store (вместо Redux)
**Преимущества:**
- ✅ Нет зависимостей
- ✅ Меньше кода (280 строк vs Redux 500+)
- ✅ Pub/Sub паттерн легко понять
- ✅ Отлично для демонстрации на собеседовании

```javascript
// Компонент слушает изменения
Store.subscribe('user', (newUser) => renderUI(newUser));

// Другой модуль изменяет state
Store.setState({ user: { name: 'Ivan', balance: 100000 } });
```

### 2. JWT в sessionStorage (не localStorage)
**Почему?**
- ✅ sessionStorage очищается при закрытии вкладки
- ✅ Недоступен XSS攻击 (в отличие от localStorage)
- ✅ Стандартная практика в банках
- ✅ Пароли никогда не хранятся!

### 3. Vanilla JavaScript (без React/Vue)
**Преимущества:**
- ✅ Демонстрирует понимание основ
- ✅ Меньше bundle размер (~5KB vs 50KB React)
- ✅ Показывает, как работают фреймворки под капотом
- ✅ Идеально для собеседования

### 4. Модульная архитектура (Services → Modules → Pages)
**Стекл:**
- Services: validator, sanitizer, api (повторно используемые)
- Modules: auth, transactions, credits (бизнес-логика)
- Pages: home, dashboard, transfers (UI интеграция)
- Store: единый источник истины
